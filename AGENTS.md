# General

- When you are writing or updating Markdown files, keep the maximum length of the line to 120 characters.

# Planning

- It’s OK to ask me more than three questions! I’ll answer as many questions as you have. I’d rather get the plan right
  from the start!
- When planning, store the plan as a Markdown file in the `.agent/local/plan/` directory. Prepend output Markdown
  filenames with a date prefix in `YYYY-MM-DD` format (e.g., `2026-02-09-libraries-resource-server`).

# Git

- When committing to the Git repository and changes are not trivial, consider updating the Changelog.

# OAuth Terminology

When the user speaks about OAuth, OIDC, and related extensions, they might use the following abbreviations:

- **AS**: Authorization Server
- **RS**: Resource Server
- **AT**: Access Token
- **ET**: Exchange Token
- **AC GT**: Authorization Code grant type
- **CC GT**: Client Credentials grant type
